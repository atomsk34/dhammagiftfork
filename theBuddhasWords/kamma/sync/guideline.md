# EBT Synchronization & Modification Guidelines

This document outlines all localized modifications added by the EBT edition of "The Buddha's Words" (A Treasury of Dhamma) and details how to synchronize changes from the upstream project while preserving our customizations.

---

## 1. Overview of EBT Enhancements

The EBT edition modifies the original site to provide:
1. **Chinese Āgama Parallels**: Embedded list of parallel sutta references linked dynamically.
2. **Bhikkhu Anālayo's Comparative Studies**: Injected links to comparative sutta studies (located under `ebt-study/`).
3. **Advanced Theme Switcher**: A 3-button theme selection grid (Light, Sepia, Dark) replacing the original binary toggle.
4. **Quick Sutta Jump Search Row**: Search/input row at the top of suttas allowing quick navigation (e.g. typing `mn153` to jump directly).
5. **Back to Top Button**: Placed at the bottom-right of pages for easier navigation.
6. **Redirection Landing Page**: `index.html` redirects automatically to `home/index.html` to load the updated landing experience.

## 2. Plain English (Human-Readable) Modification Catalog

Compared to the clean upstream `as_upstream` codebase, the EBT edition introduces these specific changes in human-readable terms:

### 1. Unique Content & Folders Added by EBT
These files and directories exist exclusively in the EBT edition and contain translation and reference studies:
- **`parallels/`**: A folder containing ~2,000 files mapping parallel sutta passages in Chinese, Sanskrit, and Tibetan.
- **`ebt-study/`**: A folder containing ~193 files containing comparative study texts for Majjhima Nikāya suttas authored by Bhikkhu Anālayo.
- **`ebt-study/māstudies/`**: A folder containing translation studies of Chinese Āgama texts.
- **`home/mitra.html`**: A custom landing homepage designed for EBT study groups.
- **Agama Parallel Scripts (`js/agama-*.js`)**: JavaScript helpers that manage comparative panels for Agama texts.
- **Index Database Scripts (`js/sutta-index.js`, `parallels-data.js`, `dhammatalks-index.js`)**: Flat databases containing dictionary pointers, parallel cross-references, and Dhammatalks.org audio link references.

### 2. Sutta HTML Page Modifications
Every single one of the 3,963 static Sutta HTML pages has been programmatically updated with EBT wrappers:
- **In the Header**: Injected a 3-button theme selection grid (**Light**, **Sepia**, **Dark**) and a **Quick Search** row (where typing a reference like `mn153` jumps you directly to that page).
- **In the Footer**: Injected a fixed **Back-to-Top** circle button.
- **Relative Paths**: Redirected all relative references pointing to `../index.html` (which upstream uses as the home page) to point to the new home landing page `../home/index.html`.
- **Scripts**: Added references to load the new EBT search bar scripts (`js/sutta-search.js`) just before the closing `</body>` tag.

### 3. CSS & Styling Modifications (`css/css.css` & `css/menu.css`)
Modified to support color themes and layout behaviors without changing the Sutta text formatting:
- **Sepia and Dark Themes**: Added color variables and background/text rules to support the Sepia (parchment color) and Dark modes.
- **Theme UI Elements**: Added classes to style the theme selection grid, the quick search input box, and the fixed-position Back-to-Top button.
- **Agama Bilingual Pages**: Configured a side-by-side split screen layout for Agama pages. It styles the Chinese column to render normally (without forcing italics) and ensures proper alignment.
- **Legacy Form Fixes**: Overrode the old hardcoded `color="#000000"` tags generated by the search forms. In dark or sepia modes, these text fields now correctly inherit the theme color instead of remaining black.
- **Top Navigation Color adjustments**: Changed the background and border colors in `css/menu.css` to match the sepia and dark themes.
- **Pāḷi & English Fonts**: Restored to use the upstream font settings exactly (no custom size reduction, no forced italics on English translations, and no forced bold on Pāḷi text).

### 4. JavaScript & Site Logic Modifications
- **Core Footer Actions (`js/footer.js`)**:
  - **`localStorage` Protection**: Wrapped all preferences storage in a safe in-memory fallback helper. If a user views the site offline (`file://` URLs) or in private mode, direct access to `localStorage` can crash the browser tab; this wrapper prevents crashes.
  - **Dynamic Linking**: Added function `injectExternalTranslationLinks()` which automatically queries the parallels database and appends direct links to alternative translations (like SuttaCentral, Thanissaro, and s.4nt.org) under the Sutta title.
  - **Anālayo Study Links**: Added function `injectComparativeStudy()` which checks if the current MN sutta has a corresponding comparative study file in `ebt-study/` and appends a link if it exists.
  - **Title flanking**: Relocated the Previous/Next Sutta links from the bottom-left corner of the page to sit directly next to the Sutta title (e.g. `< MN4: The Root of All Things >`).
  - **Keyboard Shortcuts**: Bound shortcuts (`h` for home, `?` for help, and theme toggle).
- **Control Panel (`js/controlpanel.js`)**:
  - **Path Resolution**: Updated the script to dynamically detect its own path. This prevents broken scripts (404 errors) when viewing pages in nested subdirectories (such as comparative study folders).
  - **Database Injections**: Programmatically imports the new EBT index files when the control panel loads.
  - **Random Sutta Button**: Injected a new "Random" button into the menu bar which picks a random sutta reference and redirects the browser to it.

### 5. Redirect Landing Page (`index.html`)
Upstream's default `index.html` is a full index page. EBT replaces it with a simple landing script that immediately redirects the browser to `home/index.html`, where the EBT home experience is hosted.

## 3. Comprehensive File Registry

### 3.1 Unique Folders and Files Added by EBT
These files and directories exist *only* in the EBT edition and must be preserved:
- **`parallels/`**: Directory containing ~2,000 parallel reference HTML files.
- **`ebt-study/`**: Directory containing ~193 comparative study HTML files.
- **`ebt-study/māstudies/`**: Contains Chinese Āgama translation studies.
- **`home/mitra.html`**: A custom Mitra homepage.
- **`js/agama-controlpanel.js`**, **`js/agama-en-index.js`**, **`js/agama-footer.js`**: Core script files for Chinese Āgama parallel navigation.
- **`js/dhammatalks-index.js`**: Reference mappings for dhammatalks.org links.
- **`js/parallels-data.js`**: The parallels lookup database.
- **`js/sutta-index.js`**: Index of Sutta references.
- **`js/sutta-search.js`**: Logic for the sutta-search row/jump bar.

### 3.2 Core Files Modified by EBT
These files exist in upstream but contain EBT-specific modifications. During updates, upstream changes must be merged, but EBT customizations *must* be preserved:

#### A. `index.html`
- Replaced the entire landing page content with a `<meta http-equiv="refresh" content="0; url='home/index.html'" />` redirect, along with fallback links and theme-setting JavaScript blocks.

#### B. `.gitignore`
- Added `.DS_Store` to the ignore list.

#### C. `js/footer.js`
- **Safe `localStorage` Wrapper**: Added `var storage = (function () { ... })();` shim to protect against `SecurityError` exceptions (opaque origins, private browsing, etc.) which otherwise crash subsequent script execution.
- **Dynamic External Translation Links**: Added `injectExternalTranslationLinks()` to build and insert B. Sujato, B. Thanissaro, and s.4nt.org links dynamically before `#nextprev`.
- **Dynamic Comparative Study Link**: Added `injectComparativeStudy()` to build and insert Bhikkhu Anālayo's comparative study links on Majjhima Nikāya (MN) pages.
- **Shortcut Key Bindings**: Added key shortcuts (e.g. `?` for help).

#### D. `js/controlpanel.js`
- **Self-Path Resolution**: Added dynamic URL path resolution to determine the `js/` directory location, preventing 404s when files are loaded from deeper subfolders.
- **Dynamic Database Loading**: Programmatically injects `<script src="...sutta-index.js"></script>`, `parallels-data.js`, and `dhammatalks-index.js` script tags.
- **Custom Control Panel Buttons**: Replaced the original buttons with EBT buttons, injecting the **Random Sutta** button (`#randomSutta`) and formatting the Pāli toggle button.
- **Dynamic Button Relocation**: Relocates the "Random" and "Pāḷi" buttons into the `.theme-btns` row when DOM is ready, wrapping theme buttons in `.theme-switch-group` and control buttons in `.control-group`, and removing the empty `#controlpanel` wrapper.

#### E. `css/css.css`
- **Theme Switcher Styles**: Custom CSS classes for the sepia and dark themes (`body[light-mode=dark]`, `body[light-mode=sepia]`) and style definitions for the theme buttons grid (`.theme-btns`, `.theme-btn`, `.theme-btn-light`, `.theme-btn-sepia`, `.theme-btn-dark`).
- **Relocated Buttons Layout**: Adds styles for `.theme-btns-relocated`, aligning theme and control groups horizontally (space-between) on desktop with a 25px right margin, and stacking them vertically (centered) on mobile for clear separation. Re-styles the `.cp-btn` elements inside the theme row to match `.theme-btn` size metrics (height, padding, transitions) while keeping brown colors.
- **Sutta Jump Search Box**: Styles for the search box row (`.theme-search-row`, `#sutta-search`, `#sutta-search-error`).
- **Back-to-Top Button**: Smooth scrolling (`html { scroll-behavior: smooth; }`) and styling for the fixed bottom-right button (`.back-to-top-btn`, `.back-to-top-btn svg`).
- **Chinese Āgama Bilingual Layouts**: Added `.agama-pairs` styles for layout, font sizing, line height, and explicitly overriding font style to normal (non-italic) for Chinese text.
- **Legacy Form Styling Overrides**: Replaced hardcoded black text colors in search forms with `color: inherit !important` to ensure readability in dark/sepia modes.
- **Pāḷi & English Text Preservation**: Reverted `.pairs td:last-child p` and other Sutta content styles to match `as_upstream` exactly (no custom font-family, font-style: italic, or font-size overrides on translation columns). Sutta text display is kept identical to upstream.

#### F. `css/menu.css` & `css/w3.css`
- Appended styling overrides to support the sepia and dark themes, header logo backgrounds, and button state colors.

## 4. Sutta Page Templates Modified by EBT
Sutta content HTML files (in directories `an/`, `sn/`, `mn/`, `dn/`, `ud/`, `dhp/`, `snp/`, `tha/`, `thi/`, `kp/`, `vi/`, `it/`, `kn/`, `guide/` and abbreviations pages) contain these injected blocks:
1. **Theme Switcher Grid** (located below the navigation bar):
   ```html
   <div class="theme-btns">
       <button class="theme-btn theme-btn-light" onclick="set_theme('light')" title="Light mode">Light</button>
       <button class="theme-btn theme-btn-sepia" onclick="set_theme('sepia')" title="Sepia mode">Sepia</button>
       <button class="theme-btn theme-btn-dark" onclick="set_theme('dark')" title="Dark mode">Dark</button>
   </div>
   ```
2. **Sutta Quick-Search Box** (located below the theme switcher, omitted on Vinaya and helper pages):
   ```html
   <div class="theme-search-row">
       <input id="sutta-search" type="text" placeholder="mn153, sn5.08" autocomplete="off" spellcheck="false">
       <span id="sutta-search-error"></span>
   </div>
   ```
3. **Back to Top Button** (above the footer div):
   ```html
   <div class="back-to-top-row"><a href="javascript:void(0)" onclick="window.scrollTo({top:0,behavior:'smooth'});" class="cp-btn back-to-top-btn">Back to Top</a></div>
   ```
4. **Theme Javascript Initialization & Script Reference**:
   - Uses `set_theme()` logic instead of binary `toggle_light_mode()`.
   - References `../js/sutta-search.js` just before `</body>`.

## 5. Upstream Synchronization Guide

Since upstream updates are released frequently, we maintain a clean upstream copy on the `as_upstream` branch. Use the following process to sync modifications into our working branch (`ebt` or `main`):

### Step 1: Update Upstream Branch
Ensure the `as_upstream` branch is updated to match the latest upstream release exactly using the registered `upstream` remote (`https://github.com/thebuddhaswords/BW2.git`):
```bash
# Register the upstream remote if it doesn't exist yet
git remote get-url upstream || git remote add upstream https://github.com/thebuddhaswords/BW2.git

# Fetch latest commits from upstream
git fetch upstream

# Checkout the clean upstream branch
git checkout as_upstream

# Reset local branch to match upstream/main exactly
git reset --hard upstream/main

# Push the updated branch to our origin fork
git push origin as_upstream --force
```

### Step 2: Merge Upstream into EBT
Switch to the active `ebt` branch and merge the new upstream changes.
```bash
git checkout ebt
git merge as_upstream
```

### Step 3: Resolve Conflicts
During the merge, conflicts may occur in:
- `index.html`: Keep EBT's redirect metadata.
- `.gitignore`: Retain `.DS_Store`.
- `css/` files: Preserve custom classes.
- `js/footer.js`: Retain `injectExternalTranslationLinks` and `injectComparativeStudy`.

### Step 4: Reapply HTML UI Templates
If upstream modified or added Sutta HTML files, they will lack our UI enhancements.
Run the programmatic UI script to inject headers, footers, and scripts into all sutta files on the branch:
```bash
python3 kamma/sync/scripts/apply_ebt_ui.py --all
```

### Step 5: Run Verification
Run the `check_diff.py` script to audit that all EBT enhancements are correctly injected and all assets are present:
```bash
python3 kamma/sync/scripts/check_diff.py
```
